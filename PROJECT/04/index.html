03. Problems & Solution

  • 화면 크기에 따른 페이지 대응

    모바일 화면을 제작할 때, 414px을 기준으로 두고 작업하였다. 하지만 다른 모바일 디바이스 화면에 대응이 잘되지 않아서 페이지가 넘쳤다. 그래서 medeia Query를 이용하여 max-width 값을 414px로 설정하고 각 화면의 width과 font-size를 vw로 적용했다. 그래서 다른 디바이스에서도 대응이 가능하였다.

  • react scss, JavaScript

    react를 이용하여 처음 사이트를 제작하다 보니, 기본적인 부분에서도 쉽지 않았다. scss를 처음 작업을 했을 때 중복되는 값이 너무 많아서 코드가 길어졌고, 이후에는 알아보기 어려웠다. 그래서 @mixin를 이용하여 중복되는 값을 함수로 지정 후 해당하는 class에 함수를 호출하여 적용하였다.

    JavaScript를 이용하여 동작을 구현하려고 했을 때, 이전에 사용하던 대로 적용하려 하였으나 제대로 구현되지 않았다. 그래서 react에 관하여 조금 더 방법을 찾았고, useEffect를 사용하여 스크롤과 클릭 시 변화에 대하여 적용하게 되었다.

  • build 후 빈화면 출력

    화면을 전부 제작 후 build를 했을 때 빈 화면이 출력이 되었다. build 폴더의 index.html의 link에 절대 경로로 설정이 되어있어서 출력에 오류가 났었다. 그래서 상대 경로로 바꿔주었더니 화면이 출력이 되었다.

  • image 출력 안됨

    scr 폴더에 image 파일을 저장했더니 화면에 이미지가 출력되지 않았다. 해결 방법으로 pubilc 폴더에 이미지를 옮겼고, 컴포넌트에 'process.env.PUBLIC_URL'를 추가하여 절대 경로로 경로를 수정하였다. 그리고 깃허브에 커밋 후 package.json 파일에 "homepage" 주소를 추가하였다. 이후 화면이 제대로 출력 되었다.

  • kakao map API 적용

    모바일 페이지에 kakao map API를 이용하여 지도를 적용하고 싶었다. 하지만 react에서 제대로 작동되지 않았다. 구글링을 통해 여러 가지 방법들을 찾으면서 "const { kakao } = window;" 도 넣어봤는데 제대로 작동되지 않았다. 연결의 문제인지 아직 해결법을 찾지 못하였고, API와 React에 대하여 조금 더 공부 후 적용할 예정이다.

04. Review

React를 조금 배운 후 모바일 사이트를 제작하였습니다. 처음 배우던 당시에도 이해하는 데 어려움을 느꼈습니다. 컴포넌트를 나누는 것도 어색하고 Java Script 작성도 조금 달라서 쉽지 않았습니다.

react를 이용하면서 가장 많이 했던 게 구글링이었습니다. 검색을 하며 방법을 찾는데 많은 시간이 들었습니다. 모바일 디자인도 가장 기본적인 레이아웃으로 제작해놓았고, 동작도 필요한 요소만 넣었습니다. 그러다 보니 맨 처음 HTML과 CSS를 이용하여 만들었던 페이지와 매우 유사했습니다.

아직 조금 어색한 부분들이 많고 UI/UX디자인에 있어서 아쉬운부분도 많지만 React가 어떤식으로 구동이 되고 build를 하며 출력하는 방법을 알게 되었습니다. 좀 더 공부를 한 후 동작도 더 많이 되고, API를 적용하여 구현할 수 있게 만들도록 할 것 입니다.
